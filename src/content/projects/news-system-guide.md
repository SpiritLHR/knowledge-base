这个知识库内置了一个全自动的每日新闻抓取系统。每天早 8 点和晚 6 点，系统自动从国内外新闻源拉取最新资讯，去重合并后部署上线。

## 整体流程

```
              ┌──────────────────────┐
              │   GitHub Actions     │
              │  (美国服务器, 无墙)    │
              │                      │
 定时 8:00 ──→│  1. 请求4个RSS源      │
 定时 18:00──→│  2. 解析标题+摘要      │
              │  3. 去重合并          │
              │  4. 写入.md文件        │
              │  5. git commit + push │
              └──────┬───────────────┘
                     │ 推送新闻文件
                     ▼
              ┌──────────────┐
              │  GitHub 仓库   │
              │              │
              │ 新闻文件更新   │──→ 触发 Deploy 工作流
              └──────┬───────┘        │
                     │                ▼
                     │        ┌──────────────┐
                     │        │  GitHub Pages │
                     │        │  线上站点      │
                     │        └──────────────┘
                     │
        本地 npm run fetch-news
        （仅36氪+Solidot能通）
```

## 四个新闻源

| 源 | 类型 | 本地 | Actions | 覆盖内容 |
|-----|------|------|---------|---------|
| 国内聚合 | RSSHub → 联合早报 | 需代理 | 直通 | 中国/亚洲要闻 |
| 国际聚合 | Google News | 需代理 | 直通 | 全球头条 |
| 36氪 | 直连 RSS | 直通 | 直通 | 科技商业 |
| Solidot | 直连 RSS | 直通 | 直通 | 科技资讯 |

## 为什么用两台机器

核心问题是 **GFW 网络封锁**。Google News、RSSHub 等聚合服务在国内无法直接访问，但 GitHub Actions 的运行环境在美国，不受此限制。

于是采用双轨制：

- **GitHub Actions**：主力抓取，跑在美国服务器，四个源全通。每次抓取后把结果 commit 回仓库，自动触发站点部署
- **本地 `npm run fetch-news`**：辅助手段，36氪和 Solidot 国内可直连。适合想立即看到最新内容时手动触发

无论哪边抓的，结果都汇入 `src/content/news/` 下的同一个按日期组织的 Markdown 文件，部署后站点上看到的是两边合并的完整内容。

## 技术实现

抓取脚本 `scripts/fetch-news.ts` 的核心逻辑：

1. 读取 `scripts/feeds.json` 中的 RSS 源配置
2. 用 `rss-parser` 逐个请求，提取标题、链接、摘要、来源
3. 根据 URL 去重，避免同一条新闻反复出现
4. 按日期分组写入 `src/content/news/YYYY-MM-DD.md`
5. 同一天多次运行会自动合并，不会覆盖已有内容

GitHub Actions 的定时配置在 `.github/workflows/fetch-news.yml`，使用 UTC 时间的 cron 表达式，对应北京时间 8:00 和 18:00。同时也支持手动触发。

## 如何调整新闻源

编辑 `scripts/feeds.json`，格式为：

```json
{
  "name": "源名称",
  "url": "RSS地址",
  "category": "国内新闻 或 国际新闻"
}
```

修改后推送即可，下次抓取自动生效。也支持通过对话告诉我直接改。
